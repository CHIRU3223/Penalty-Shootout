# AI System

Configurable AI for solo mode (keeper + shooter).

## Location

| Package | File |
|---------|------|
| Core logic | `shared/src/ai.ts` |
| Difficulty configs | `shared/src/difficulty.ts` |
| Client wrapper | `client/src/game/ai.ts` |
| Tests | `shared/src/ai.test.ts` |

## Usage (client)

```typescript
import { createAiOpponent } from './game/ai';

const opponent = createAiOpponent('intermediate');

// When player shoots — AI picks dive
const dive = opponent.pickDive(shot, aimTellZone);

// When AI shoots — AI picks shot
const shot = opponent.pickShot();
```

## Difficulty configuration

```typescript
interface DifficultyConfig {
  saveChance: number;      // Probability keeper dives to correct zone
  saveRadius: number;      // Grid distance for save overlap (Chebyshev)
  reactionMs: number;      // Reaction window (used in aim-tell reading)
  shooterAccuracy: number;   // Corner placement + power consistency
}
```

| Level | saveChance | saveRadius | reactionMs | shooterAccuracy |
|-------|------------|------------|------------|-----------------|
| **Beginner** | 0.20 | 0 | 600 | 0.35 |
| **Intermediate** | 0.45 | 1 | 400 | 0.60 |
| **Pro** | 0.70 | 1 | 200 | 0.85 |

## Keeper AI (`pickKeeperDive`)

Decision order:

1. If `aimTell` provided and RNG < `shooterAccuracy × 0.85` → dive to aim tell zone
2. Else if RNG < `saveChance` → dive to shot zone
3. Else → dive to random neighbor zone
4. Optional: with `saveRadius > 0`, sometimes dive adjacent for coverage

**Beginner (~20% correct):** mostly random neighbors  
**Pro (~70% correct):** reads shot zone + aim tell, larger effective coverage

## Shooter AI (`pickShooterShot`)

Zone selection:

| RNG vs accuracy | Zone picked |
|-----------------|-------------|
| < accuracy × 0.75 | Random corner (0, 2, 6, 8) |
| < accuracy | Random side (1, 3, 5, 7) |
| < 0.5 | Center (4) |
| else | Random any zone |

Power: `0.35 + accuracy × 0.45` ± small random variance, capped at 0.98.

**Beginner:** center/sides, lower power  
**Pro:** corners, high power

## Tuning

Edit values in `shared/src/difficulty.ts`. Both client AI and server online difficulty use `getDifficultyConfig()` (server currently uses `intermediate` for online).

## Tests

```bash
npm test -w shared -- ai
```

Covers:
- Beginner correct dive with low RNG
- Beginner miss with high RNG
- Pro reads aim tell
- Beginner lower power vs Pro higher power
